# Security Policy

## Reporting a vulnerability

Until a dedicated security contact is published, report suspected
vulnerabilities privately to `0atxlk@gmail.com`. Do not include sensitive paste
contents or encryption keys in public issues.

Please include a clear reproduction, affected revision, and security impact.
We will acknowledge reports and coordinate a fix before public disclosure.

## Supported versions

Before the first release, security fixes are made on the `main` branch.
